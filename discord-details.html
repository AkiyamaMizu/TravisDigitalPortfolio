Problem Statement 4B

Apex Legend: Discord Intelligence

The Chaos of "Data Doom-Scrolling": In the IT Consulting sector, vital project files are often lost in the noise of endless chat histories. Teams waste hours scrolling to find a single PDF or Excel sheet, leading to missed deadlines and frustration.

Our Solution: We built an AI-Powered Discord Ecosystem. Instead of forcing users to switch platforms, we deployed a silent "Intelligence Bot" that automatically intercepts, categorizes, and archives shared files into a secure central database. It transforms a chaotic chat app into a structured, searchable document management system.

My Contribution: The Intelligence Bot

I served as the Lead Bot Engineer. My responsibility was to build the Python-based intelligence layer that sits between the user and the database, focusing on Security, Automation, and Reporting.

01. Dynamic Collection Control

A core requirement was user consent and controlâ€”we didn't want a "spy bot" that runs forever. I implemented a strict Collection Lifecycle System.

Enabling: Admins can trigger `/enable_collection` to spin up the background listening threads. The bot confirms activation immediately with a success embed (`acollectionenable.png`).

Status Checks: At any time, staff can verify the bot's heartbeat using `/collector_status`. This provides instant feedback (`activecollectionstatus.png`), ensuring the team knows exactly when their data is being archived and when the system is dormant.

02. Malware Defense Protocol

Security was our #1 barrier. We couldn't risk users uploading malicious scripts. I implemented a strict File Whitelist Protocol directly in the bot's event listener.

As shown here, when I attempted to upload a file named `virus.exe`, the bot immediately intercepted the packet. It rejected the file and issued a public Security Alert embed. This "Zero-Trust" approach ensures that only safe assets (PDFs, Images, Excel) ever reach our database.

03. Backend Threat Surveillance

Security isn't just about blocking; it's about tracking. I programmed the bot to log every security incident directly to the server console.

This screenshot shows the Terminal View during an attack. You can see the system explicitly logging: [SECURITY BLOCK] Prevented Travis from uploading: virus.exe. This gives our DevSecOps team real-time visibility into attempted breaches.

04. Real-Time Asset Reporting

To solve the "Disorganized Data" problem, I built the `!generate_report` slash command. This tool provides admins with an instant snapshot of the department's data health.

The bot queries the SQLite database in real-time and returns a Department Activity Report. It breaks down collected assets by type (Images vs. PDFs), proving that the bot is actively sorting chaos into order without human intervention.

05. Role-Based Access Control

A powerful tool shouldn't be accessible to everyone. I implemented RBAC decorators on sensitive commands.

Here, we see a standard user (non-admin) attempting to tamper with the bot's settings. The bot verifies the user's role against the server's permission hierarchy and immediately denies the request. This ensures that only authorized Project Managers can control the data flow.

06. DevOps & Maintenance

Maintaining a bot requires robust utility commands. I implemented a Command Sync feature (`syncedcommands.png`) to instantly push new Slash Command updates to the Discord API without restarting the server.

Additionally, the Graceful Shutdown protocol (`shutdown.png`) ensures that when the bot is taken offline for maintenance, all database connections are closed safely, preventing data corruption.
